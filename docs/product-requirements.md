# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**迷宮ラン** - スマホで遊べるパックマン風迷路ゲーム

### プロダクトコンセプト
- **ゼロ依存・即プレイ**: Vanilla JS + Canvas API のみで動作し、インストール不要。GitHub Pages で URL を共有するだけで誰でも即プレイ可能。
- **スマホファースト操作**: スワイプ操作による直感的なモバイル対応。壁への衝突時にその場でピタッと停止するレスポンシブな操作感。
- **オリジナル著作権対応**: タイトル・素材はすべてオリジナル制作。ゲームメカニクスのみ参考にし、商標・素材の無断使用を排除。

### プロダクトビジョン
「迷宮ラン」は、誰もが知るアーケードゲームのエッセンスを大切にしながら、スマートフォン時代に最適化したブラウザゲームを目指す。
GitHub Pages で無料公開し、URL を共有するだけで友人・家族が即座に楽しめる。
ビルドツール・依存ライブラリゼロの構成により、長期的なメンテナンス性を確保し、誰でも気軽に改造・貢献できるオープンなプロジェクトとする。

### 目的
- ブラウザ単体で動作するパックマン風ゲームを GitHub Pages で無料公開する
- スマートフォン・PC の両方でストレスなくプレイできる操作性を実現する
- 著作権リスクを排除した上でゲームメカニクスを忠実に再現する
- PWA 化によりスマホのホーム画面からアプリとして起動できるようにする

## 成功指標(KPI)

### プライマリーKPI
- **プレイ開始率**: GitHub Pages にアクセスしたユーザーの 80% 以上がゲームを開始する（公開後 1 ヶ月）
- **1 セッション平均プレイ時間**: 3 分以上（公開後 1 ヶ月）
- **モバイル操作満足度**: スワイプ操作で意図した方向に 95% 以上の確率で動く（手動テストによる検証）

### セカンダリーKPI
- **GitHub Stars**: 公開後 3 ヶ月で 50 Stars 以上
- **PWA インストール率**: モバイル訪問者の 10% 以上がホーム画面追加を実行
- **ゲームオーバーまでのプレイ継続率**: 初回プレイで 1 ステージをクリアまたはゲームオーバーになるまで 90% 以上が離脱しない

## 機能要件

### コア機能（フェーズ1 MVP）

#### 1. ゲームフィールド描画

**ユーザーストーリー**:
プレイヤーとして、原作を彷彿とさせる迷路の中でゲームをプレイするために、正確なマップ描画が欲しい

**受け入れ条件**:
- [ ] Canvas API を使用してマップを描画し、60fps を維持する
- [ ] 壁・通路・ドット・パワーエサを色分けして表示する
- [ ] スマホ画面サイズ（375px 幅を基準）にフィットするレスポンシブレイアウト
- [ ] PC ブラウザでも同様に表示できる

**優先度**: P0（必須）

---

#### 2. プレイヤー操作

**ユーザーストーリー**:
プレイヤーとして、直感的に操作するために、スワイプおよびキーボード入力で迷路内を移動したい

**受け入れ条件**:
- [ ] スワイプ操作（touchstart / touchend）で上下左右 4 方向の移動ができる
- [ ] スワイプ距離 30px 未満は誤検知として無視する
- [ ] 壁がある方向にスワイプした場合、その位置でピタッと停止する（バッファリングなし）
- [ ] `preventDefault()` によりスワイプ中のページスクロールを抑制する
- [ ] PC ではキーボード（矢印キー / WASD）で操作できる
- [ ] 方向入力は次のタイル境界で反映される（タイル単位の移動制御）

**優先度**: P0（必須）

---

#### 3. ドット・パワーエサの収集

**ユーザーストーリー**:
プレイヤーとして、スコアを稼ぐために、迷路内のドットとパワーエサを食べたい

**受け入れ条件**:
- [ ] ドット（小）を食べると 10 点加算される
- [ ] パワーエサ（大）を食べると 50 点加算され、イジケ状態（一定時間ゴーストを食べられる）に移行する
- [ ] 食べたドット・パワーエサはフィールドから消える
- [ ] 全ドットを食べるとステージクリアになる

**優先度**: P0（必須）

---

#### 4. ゴーストAI（4体）

**ユーザーストーリー**:
プレイヤーとして、戦略的に逃げたり攻めたりするために、それぞれ異なる行動パターンを持つゴーストと対戦したい

**受け入れ条件**:
- [ ] **Blinky（赤）**: プレイヤーの現在位置を直接ターゲットに追跡する
- [ ] **Pinky（ピンク）**: プレイヤーの進行方向 4 タイル先をターゲットにする
- [ ] **Inky（青）**: Blinky の位置とプレイヤーの進行方向 2 タイル先の対称点をターゲットにする
- [ ] **Clyde（オレンジ）**: プレイヤーとの距離が 8 タイル以上なら追跡、8 タイル未満なら左下コーナーへ逃げる
- [ ] ゴーストはスタートエリア（ゴーストハウス）から一定間隔で出現する
- [ ] イジケ状態（パワーエサ後）ではゴーストが青く点滅し、プレイヤーが食べられる
- [ ] イジケ状態でゴーストを食べると 200/400/800/1600 点と連鎖ボーナスが付く

**優先度**: P0（必須）

---

#### 5. スコアシステム

**ユーザーストーリー**:
プレイヤーとして、プレイの達成感を感じるために、スコアとハイスコアを確認したい

**受け入れ条件**:
- [ ] 現在スコアを画面上部にリアルタイム表示する
- [ ] ハイスコアを `localStorage` に保存し、セッションをまたいで永続化する
- [ ] ゲームオーバー・ステージクリア時にハイスコアを更新する

**優先度**: P0（必須）

---

#### 6. ゲームサイクル（残機・ゲームオーバー）

**ユーザーストーリー**:
プレイヤーとして、緊張感を持ってプレイするために、残機制とゲームオーバーが欲しい

**受け入れ条件**:
- [ ] 初期残機は 3 機
- [ ] ゴーストに触れると 1 機失い、スタート位置から再開する
- [ ] 残機が 0 になるとゲームオーバー画面を表示し、スコアを確定する
- [ ] ゲームオーバー後にリスタートできる

**優先度**: P0（必須）

---

#### 7. BGM・効果音

**ユーザーストーリー**:
プレイヤーとして、ゲームの雰囲気を楽しむために、BGM と効果音が欲しい

**受け入れ条件**:
- [ ] ゲーム中の BGM を再生する（フリー素材使用）
- [ ] ドット収集・パワーエサ収集・ゴースト撃退・プレイヤー死亡の効果音を再生する
- [ ] 音量調整 or ミュートボタンを設ける
- [ ] ブラウザの自動再生ポリシーに対応し、ユーザーインタラクション後に BGM を開始する

**優先度**: P0（必須）

---

#### 8. PWA 対応

**ユーザーストーリー**:
スマホユーザーとして、アプリのように使うために、ホーム画面に追加してオフラインで起動したい

**受け入れ条件**:
- [ ] `manifest.json` を設置し、アプリ名・アイコン・テーマカラーを定義する
- [ ] `service-worker.js` でゲームアセットをキャッシュしオフライン対応する
- [ ] スマホブラウザで「ホーム画面に追加」プロンプトが表示される
- [ ] ホーム画面からフルスクリーンで起動できる

**優先度**: P0（必須）

## 非機能要件

### パフォーマンス
- ゲームループ: 60fps を安定維持（Chrome / Safari / Firefox 最新版で検証）
- 初回ロード時間: 3G 相当（3Mbps）の環境で 3 秒以内にゲーム開始可能な状態にする
- タッチ入力レイテンシ: `touchend` から方向転換反映まで 1 フレーム（16ms）以内

### ユーザビリティ
- 操作説明を見なくても 1 プレイ以内に基本操作を習得できる（スワイプのみで操作完結）
- スマホ縦向き（Portrait）・横向き（Landscape）どちらでも正常にプレイできる
- 画面幅 320px 〜 428px の主要スマホサイズで崩れなく表示される

### 信頼性
- ハイスコアは `localStorage` に書き込み、ブラウザリロード後も保持される
- ゴーストの AI ロジックがデッドロック（無限ループ）に陥らない
- ゲームループのエラーで画面が固まった場合、リスタートボタンで復帰できる

### セキュリティ
- 外部 API・サーバー通信なし（localStorage のみ）のため攻撃面が最小
- `Content-Security-Policy` ヘッダーを設定し XSS リスクを軽減する（GitHub Pages 対応範囲内）
- 使用するフリー素材のライセンス（CC0 / MIT 等）を `LICENSE` またはクレジットファイルで明記する

### スケーラビリティ
- 静的ファイルのみで構成し、GitHub Pages の CDN に乗せることで無制限のアクセスに対応
- JavaScript は単一バンドルファイルを避けモジュール分割し、将来的な機能追加を容易にする

## スコープ外

明示的にスコープ外とする項目:
- 「PAC-MAN」「パックマン」商標の使用
- オリジナル Namco の素材・音源の流用
- バックエンドサーバー・データベースの構築
- ユーザー認証・アカウント管理
- 広告・課金機能
- ネイティブアプリ（iOS / Android）の配布
