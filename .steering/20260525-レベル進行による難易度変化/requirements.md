# 要求内容

## 概要

ステージクリア後に次のレベルへ進むたびに、ゴーストの速度・プレイヤー速度・イジケ時間・ゴーストリリース閾値などのゲームパラメータを段階的に調整し、難易度を上げる機能を実装する。

## 背景

現在、ステージをクリアしても次レベルへのパラメータ変化がなく、何周しても同じ難易度でリプレイ性が低い。オリジナルPAC-MANに倣い、レベルが上がるにつれてゲームが難しくなることでプレイヤーの挑戦意欲とリプレイ性を高める。

## 実装対象の機能

### 1. レベル別パラメータ定義

各レベルに応じたゲームパラメータ（速度・イジケ時間・ゴースト出現タイミング）をデータとして定義する。

- `LevelParams` 型の定義
- `getLevelParams(level: number)` 関数の実装（Level 5以上は上限値でキャップ）

### 2. GhostManagerのレベル対応

ゴーストの移動速度・イジケ速度・イジケ持続時間・ゴーストリリース閾値をレベルパラメータから取得するよう変更する。

- `reset(params: LevelParams)` でレベルに応じたパラメータを注入
- インスタンス変数として速度・持続時間を保持し、定数を直接参照しない

### 3. PlayerManagerのレベル対応

プレイヤーの移動速度をレベルパラメータから取得するよう変更する。

- `reset(speed: number)` でレベルに応じた速度を注入
- インスタンス変数として速度を保持

### 4. GameLoopのレベルパラメータ適用

ステージクリア・リスポーン・新ゲーム開始時に現在レベルのパラメータを各マネージャへ渡す。

## 受け入れ条件

### レベルパラメータ

- [ ] Level 1〜5以上で異なるパラメータが適用される
- [ ] Level 5以上は同一パラメータ（上限キャップ）
- [ ] ゴーストはプレイヤーより常に遅いかほぼ同速（Level 5でもプレイヤーが逃げ切れる余地がある）

### 難易度変化

- [ ] レベルが上がるにつれゴーストが速くなる
- [ ] レベルが上がるにつれイジケ時間が短くなる
- [ ] レベルが上がるにつれゴーストが早くゴーストハウスから出てくる
- [ ] プレイヤー速度もわずかに上がり、高レベルでも操作感が維持される

### コード品質

- [ ] 既存の定数（`GHOST_SPEED`, `FRIGHTENED_SPEED` 等）はLevel 1のデフォルト値として残す
- [ ] TypeScript 型エラーなし
- [ ] ビルド成功

## 成功指標

- Level 1: 現行と同じ難易度（プレイヤーが十分逃げられる）
- Level 3以上: 明らかにゴーストが速く緊張感が増す
- Level 5以上: 高難度だが理不尽ではない（プレイヤー速度 > ゴースト速度を維持）

## スコープ外

以下はこのフェーズでは実装しません:

- フルーツボーナスアイテムの追加
- レベル数の画面表示デザイン変更（既存の「LV.X」表示で十分）
- BGM/SEの変更
- マップレイアウトのレベル差分
- エクストラライフ機能
