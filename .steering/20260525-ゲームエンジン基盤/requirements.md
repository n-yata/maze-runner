# 要求定義: ゲームエンジン基盤

## 概要

迷宮ランのゲームエンジン基盤を構築する。プレイヤー・ゴースト・マップ・レンダリング・入力・音声・スコア管理の全コアモジュールを実装し、ゲームが起動してプレイできる状態にする。

## 背景・目的

- プロジェクトは現在ソースコードが存在しない状態（greenfield）
- 最初の `/add-feature` として、全ての機能の土台となるゲームエンジンを構築する
- この基盤の上に、今後の機能追加・改善が行われる

## スコープ

### In Scope

- プロジェクトのスキャフォールディング（package.json, tsconfig.json, index.html）
- 型定義・定数の整備（src/types.ts, src/constants.ts）
- マップシステム（28×31タイルのPAC-MANマップ、タイル判定）
- ゲームループ（requestAnimationFrame + 固定1/60sタイムステップ）
- レンダラー（Canvas 2D API、OffscreenCanvasキャッシュ）
- プレイヤー制御（キーボード + スワイプ入力、移動・食べ物収集）
- ゴーストAI（Blinky/Pinky/Inky/Clyde の4体、モード切替スケジュール）
- 音声システム（Web Audio API、効果音生成）
- スコア・ハイスコア管理（localStorage）
- メインエントリポイント（src/main.ts）
- PWAファイル（manifest.json, service-worker.js）
- ユニットテスト（Vitest）

### Out of Scope

- ハイスコアランキング（複数ユーザー）
- マルチプレイヤー
- BGM（効果音のみ）
- レベルエディタ
- PWA以外のプラットフォーム対応

## 受け入れ条件

- [ ] `npm run build` でエラーなくビルドできる
- [ ] `npm test` で全テストがパスする
- [ ] `npm run typecheck` でTypeScriptエラーがない
- [ ] ブラウザでindex.htmlを開くとゲームが起動する
- [ ] キーボード操作でプレイヤーが動く
- [ ] ゴースト4体が動き、プレイヤーを追いかける
- [ ] エサを食べるとスコアが加算される
- [ ] パワーエサを食べるとゴーストがびびりモードになる
- [ ] ライフが0になるとゲームオーバー
- [ ] 全エサを食べるとステージクリア
- [ ] ハイスコアがlocalStorageに保存・表示される
- [ ] スマートフォンでスワイプ操作ができる

## 制約・前提

- 外部ライブラリ・バンドラー不使用（ゼロランタイム依存）
- TypeScript 5.x + tsc でコンパイル（ESModules、dist/配下に出力）
- GitHub Pages でホスト可能な静的ファイル構成
- 60fps を維持すること
